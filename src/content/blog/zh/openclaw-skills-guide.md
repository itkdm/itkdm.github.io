## OpenClaw 技能太多不会选？这 15 个核心技能让你效率翻倍

OpenClaw 的技能市场已经有上万个技能，新手看到这么多选择很容易懵。

其实真正好用的技能没那么多。我用了大半年，整理出 15 个最高频、最实用的核心技能，覆盖日常工作的 80% 场景。

今天就来聊聊这些技能怎么用，以及几个实战工作流，让你快速上手 OpenClaw。

---

## 一、为什么技能在精不在多？

刚开始用 OpenClaw 的时候，我也恨不得把所有技能都装上。结果发现：

- 装了一堆，真正用到的没几个
- 技能太多，反而找不到想要的
- 有些功能重复，浪费时间

后来我换了个思路：**按场景选技能，而不是按数量装技能**。

现在我的技能列表保持在 20 个以内，每个都有明确的使用场景，效率反而更高了。

---

## 二、15 个核心技能清单

### 📝 内容创作类（5 个）

**1. wechat-ai-publisher** - 公众号自动发布
- 用途：Markdown 写文章，一键发布到公众号草稿箱
- 场景：自媒体运营、内容创作者
- 亮点：自动生成封面图、支持 Mermaid 图表

**2. feishu-doc** - 飞书文档操作
- 用途：读写飞书文档、创建表格、插入图片
- 场景：团队协作、知识管理
- 亮点：支持完整文档 API，可自动化

**3. feishu-bitable** - 飞书多维表格
- 用途：创建和管理飞书多维表格
- 场景：项目管理、数据追踪
- 亮点：可创建字段、记录，支持复杂数据结构

**4. tts** - 文字转语音
- 用途：将文本转换为语音输出
- 场景：语音播报、有声内容
- 亮点：支持多种语音风格

**5. canvas** - 画布控制
- 用途：在画布上展示 UI、截图、交互
- 场景：演示、可视化展示
- 亮点：支持导航、评估、快照

### 🔍 信息获取类（4 个）

**6. web_search** - 网络搜索
- 用途：使用 Brave Search API 搜索网络
- 场景：资料搜集、信息检索
- 场景：研究、写作素材收集
- 亮点：支持区域过滤、时间过滤

**7. web_fetch** - 网页抓取
- 用途：提取网页可读内容（HTML → Markdown）
- 场景：文章收藏、内容分析
- 亮点：自动提取正文，过滤广告

**8. browser** - 浏览器控制
- 用途：自动化浏览器操作（点击、输入、截图）
- 场景：复杂网页交互、UI 测试
- 亮点：支持 Playwright，可处理动态页面

**9. memory_search** - 记忆搜索
- 用途：搜索 MEMORY.md 和记忆文件
- 场景：查找历史信息、回顾决策
- 亮点：语义搜索，支持分数过滤

### 🛠️ 系统工具类（4 个）

**10. exec** - 执行命令
- 用途：运行 Shell 命令
- 场景：服务器管理、文件操作
- 亮点：支持后台运行、PTY 模式

**11. read/write/edit** - 文件操作
- 用途：读写和编辑文件
- 场景：代码编写、配置修改
- 亮点：精确编辑，支持大文件分页

**12. process** - 进程管理
- 用途：管理后台执行的进程
- 场景：长时间任务监控
- 亮点：支持日志、输入、终止

**13. sessions_spawn** - 子代理生成
- 用途：生成隔离的子代理会话
- 场景：复杂任务分解、并行处理
- 亮点：支持子代理和 ACP 运行时

### 📱 消息通知类（2 个）

**14. message** - 消息发送
- 用途：发送消息到各种渠道（Telegram、Discord 等）
- 场景：自动通知、群发消息
- 亮点：支持多平台、富媒体

**15. feishu_chat** - 飞书聊天
- 用途：飞书群聊操作
- 场景：团队通知、群管理
- 亮点：支持成员管理、群信息

## 三、3 个实战工作流

### 工作流 1：公众号自动发布

```
1. 用 wechat-ai-publisher 创作文章
   → 自动搜索素材、撰写内容

2. 自动生成封面图
   → 调用 generate-cover.py

3. 一键发布到草稿箱
   → 调用 publish:enhanced

4. 飞书通知
   → 发送发布成功消息
```

**适用场景**：自媒体运营、内容创作者

**效率提升**：从 2 小时缩短到 15 分钟

---

### 工作流 2：信息搜集 + 整理

```
1. 用 web_search 搜索主题资料
   → 获取 10 个相关网页

2. 用 web_fetch 提取网页内容
   → 转为 Markdown 格式

3. 用 feishu-doc 创建整理文档
   → 自动归类、添加标签

4. 用 memory_search 保存关键信息
   → 方便后续检索
```

**适用场景**：研究报告、竞品分析、学习总结

**效率提升**：从 1 天缩短到 2 小时

---

### 工作流 3：服务器监控 + 通知

```
1. 用 exec 执行监控命令
   → 检查 CPU、内存、磁盘

2. 用 read 读取日志文件
   → 分析异常情况

3. 用 message 发送告警通知
   → 推送到 Telegram/飞书

4. 用 sessions_spawn 生成修复任务
   → 自动处理常见问题
```

**适用场景**：运维监控、自动化运维

**效率提升**：7×24 小时自动监控，无需人工值守

## 四、如何选择适合自己的技能？

**第一步：明确你的核心场景**

问自己三个问题：
1. 我每天花时间最多的是什么工作？
2. 哪些任务可以自动化？
3. 我希望 AI 帮我解决什么问题？

**第二步：按优先级安装**

- **P0**：每天都要用的（如文件操作、搜索）
- **P1**：每周用几次的（如公众号发布、文档管理）
- **P2**：偶尔用到的（如浏览器自动化、复杂任务分解）

**第三步：边用边优化**

- 用了两周后发现不合适的，果断卸载
- 发现新需求的，再去技能市场找
- 定期回顾技能列表，保持精简

## 五、避坑指南

**❌ 不要一次性装太多技能**

- 装 100 个技能，用到的可能只有 10 个
- 技能太多反而增加选择成本

**✅ 建议**：先装 5-10 个核心技能，用熟了再扩展

---

**❌ 不要忽视文档**

- 每个技能的 SKILL.md 都要看
- 了解使用场景和限制

**✅ 建议**：安装前先读文档，避免装错

---

**❌ 不要盲目追求新功能**

- 新功能不一定适合你
- 稳定可靠比新颖更重要

**✅ 建议**：优先选择经过验证的成熟技能

---

## 写在最后

技能只是工具，真正的效率提升来自于**用好工具的方法**。

我的建议是：

1. 先掌握这 15 个核心技能
2. 结合自己的工作场景实践
3. 逐步形成自己的工作流
4. 再根据需要扩展其他技能

记住：**技能在精不在多，用好一个胜过安装十个**。
